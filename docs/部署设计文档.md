## 1. 部署架构

### 1.1 整体流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    本地开发环境                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  make build ──→ cargo build --release                          │
│       │                                                           │
│  make package ──→ 打包成 tar.gz                                  │
│       │                                                           │
│  remote-deploy.sh ──→ scp 到远端服务器                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    远端服务器                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  tar -xzf lingbase-{v}-{arch}.tar.gz                           │
│  chmod +x lingbase run.sh                                      │
│  可选: 安装 systemd service                                      │
│  ./run.sh ──→ 启动服务                                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 部署脚本

| 脚本 | 说明 |
|------|------|
| `scripts/run.sh` | 启动脚本，自动检测架构和 CUDA 可用性 |
| `scripts/remote-deploy.sh` | 远程一键部署脚本 (编译+打包+部署) |
| `scripts/lib/deploy.sh` | 部署公共函数库 |

### 2.1 run.sh 启动脚本

核心逻辑：
1. 检测系统架构: `ARCH=$(uname -m)` → `x86_64` 或 `aarch64`
2. 检测 CUDA 可用性: `BACKEND=$(detect_cuda)` → `cuda` 或 `cpu`
3. 设置 `LD_LIBRARY_PATH` 到对应库目录
4. 验证 `libllama.so` 存在
5. 执行 `lingbase` 二进制

### 2.2 remote-deploy.sh 远程部署脚本

**do_build() 构建方式:**

直接调用 `cargo build`，不经过 Makefile：

| 条件 | 构建命令 |
|------|---------|
| `arch == aarch64` | `cargo build --release --target aarch64-unknown-linux-gnu` |
| `HAS_CUDA == true` | `cargo build --release --features cuda` |
| 其他 | `cargo build --release` |

**支持功能：**
- `-b, --build`: 仅编译
- `-p, --package`: 编译+打包
- `-d, --deploy`: 仅部署已有包
- `--uninstall`: 卸载远端服务
- `--list`: 列出可用服务器

## 3. 服务器配置

### 3.1 deploy.toml 配置

`config/deploy.toml` 定义部署目标：

```toml
[remote.gpu-server]
host = "67.0.0.5"
user = "etsme"
port = 22
password = "l7OiD3QuWNI#sC"
description = "RTX 4090 GPU Server"
arch = "x86_64"
lib_dir = "x86_64-cuda"

[remote.arm-server]
host = "192.168.0.124"
user = "firefly"
port = 22
password = "firefly"
description = "ARM64 Server"
arch = "aarch64"
lib_dir = "aarch64-rk-cpu"
```

**配置字段说明：**

| 字段 | 说明 |
|------|------|
| `host` | 服务器 IP 地址 |
| `user` | SSH 用户名 |
| `port` | SSH 端口 (默认 22) |
| `password` | SSH 密码 |
| `arch` | 目标架构 (`x86_64` 或 `aarch64`) |
| `lib_dir` | 动态库目录 (相对于 `lib/` 目录) |

### 3.2 软链接机制

当 `lib_dir` 与 `arch` 不一致时，打包时会创建软链接：

```
# 示例：arm-server 配置
arch = "aarch64"
lib_dir = "aarch64-rk-cpu"

# 打包后结构
lib/
└── aarch64/     # 软链接 → aarch64-rk-cpu/
```

`run.sh` 通过 `ARCH=$(uname -m)` 查找 `lib/aarch64/`，如果是软链接则自动解析到实际库目录。

## 4. 部署命令

### 4.1 常用命令

```bash
# 一键部署到 arm-server (编译+打包+部署)
./scripts/remote-deploy.sh arm-server

# 仅部署已有包
./scripts/remote-deploy.sh arm-server -d

# 列出可用服务器
./scripts/remote-deploy.sh --list

# 卸载
./scripts/remote-deploy.sh arm-server --uninstall
```

### 4.2 本地打包命令

```bash
# 打包当前平台
make package

# 打包 x86_64 CPU 版
make package-x86_64

# 打包 x86_64 CUDA 版
make package-x86_64-cuda

# 打包 ARM64 版
make package-aarch64

# 打包所有架构版本
make package-all
```

## 5. x86-server 部署流程详解

以部署到 x86 服务器 (`67.0.0.2`) 为例：

### Step 1: 加载配置

从 `config/deploy.toml` 读取 `x86-server` 配置：
- `host = 67.0.0.2`
- `user = etsme`
- `arch = x86_64`
- `lib_dir = x86_64-cpu`

### Step 2: 检测远端环境

```bash
# SSH 检测架构
ssh etsme@67.0.0.2 "uname -m"  # → x86_64

# 检测 CUDA
ssh etsme@67.0.0.2 "nvidia-smi -L"  # → 无 GPU → CUDA=false
```

### Step 3: CPU 构建

```bash
# 直接使用 cargo 构建 (不使用 Makefile)
cargo build --release
```

产物: `target/release/lingbase` (普通 CPU 版)

### Step 4: 打包

```
lingbase-{version}-x86_64/
├── lingbase              # x86_64 CPU 二进制
├── run.sh               # 启动脚本
├── config/
│   └── environment.toml # 配置文件
└── lib/
    └── x86_64-cpu/       # CPU 动态库
        ├── libllama.so
        ├── libggml-base.so
        └── ...
```

> 注：`lib_dir` 和 `arch` 都是 x86_64 体系，打包时不创建软链接，run.sh 直接使用 `lib/x86_64-cpu/`。

打包命令: `tar -czf dist/lingbase-{v}-x86_64.tar.gz`

### Step 5: 部署

```bash
# 上传 tar.gz
scp dist/lingbase-{v}-x86_64.tar.gz etsme@67.0.0.2:/home/etsme/lingbase/

# 远端执行
ssh etsme@67.0.0.2 << 'EOF'
cd /home/etsme/lingbase
tar -xzf lingbase-{v}-x86_64.tar.gz
chmod +x lingbase run.sh
# 可选: 安装 systemd service
EOF
```

### Step 6: 运行时

`run.sh` 启动流程：
1. 检测架构: `ARCH=$(uname -m)` → `x86_64`
2. 检测 CUDA: `BACKEND=$(detect_cuda)` → `cpu` (无 GPU)
3. 设置 `LD_LIBRARY_PATH=lib/x86_64-cpu`
4. 执行 `exec ./lingbase` (CPU 推理)

## 7. arm-server 部署流程详解

以部署到 ARM 服务器 (`192.168.0.124`) 为例：

### Step 1: 加载配置

从 `config/deploy.toml` 读取 `arm-server` 配置：
- `host = 192.168.0.124`
- `user = firefly`
- `arch = aarch64`
- `lib_dir = aarch64-rk-cpu`

### Step 2: 检测远端环境

```bash
# SSH 检测架构
ssh firefly@192.168.0.124 "uname -m"  # → aarch64

# 检测 CUDA
ssh firefly@192.168.0.124 "nvidia-smi -L"  # → 无 GPU
```

### Step 3: 交叉编译

```bash
# 安装交叉编译工具链 (如果不存在)
sudo apt-get install -y gcc-aarch64-linux-gnu

# 添加 Rust 目标
rustup target add aarch64-unknown-linux-gnu

# 交叉编译
CARGO_TARGET_AARCH64_UNKNOWN_LINUX_GNU_LINKER=aarch64-linux-gnu-gcc \
  cargo build --release --target aarch64-unknown-linux-gnu
```

产物: `target/aarch64-unknown-linux-gnu/release/lingbase`

### Step 4: 打包

```
lingbase-{version}-aarch64/
├── lingbase              # ARM64 二进制
├── run.sh               # 启动脚本
├── config/
│   └── environment.toml # 配置文件
└── lib/
    └── aarch64/         # 软链接 → aarch64-rk-cpu/
        └── ...          # 实际库文件在目标目录
```

**软链接机制:**
- `deploy.toml` 中 `lib_dir = aarch64-rk-cpu` 指定实际库目录
- 打包时创建软链接: `lib/aarch64/ -> aarch64-rk-cpu/`
- `run.sh` 通过 `ARCH=$(uname -m)` 查找 `lib/aarch64/`，自动解析到实际库目录

打包命令: `tar -czf dist/lingbase-{v}-aarch64.tar.gz`

### Step 5: 部署

```bash
# 上传 tar.gz
scp dist/lingbase-{v}-aarch64.tar.gz firefly@192.168.0.124:/home/firefly/lingbase/

# 远端执行
ssh firefly@192.168.0.124 << 'EOF'
cd /home/firefly/lingbase
tar -xzf lingbase-{v}-aarch64.tar.gz
chmod +x lingbase run.sh
# 可选: 安装 systemd service
EOF
```

### Step 6: 运行时

`run.sh` 启动流程：
1. 检测架构: `ARCH=$(uname -m)` → `aarch64`
2. 检测 CUDA: `BACKEND=$(detect_cuda)` → `cpu` (无 GPU)
3. 设置 `LD_LIBRARY_PATH=lib/aarch64`
4. 执行 `exec ./lingbase`

## 8. 打包结构

| 服务器 | arch | lib_dir | 后端 | 构建命令 | 打包目录 |
|--------|------|---------|------|---------|---------|
| x86-server | x86_64 | x86_64-cpu | CPU | `cargo build --release` | `lib/x86_64-cpu/` |
| gpu-server | x86_64 | x86_64-cuda | CUDA | `cargo build --features cuda` | `lib/x86_64-cuda/` |
| arm-server | aarch64 | aarch64-rk-cpu | CPU | 交叉编译 | `lib/aarch64/` → 软链接 |

**说明:**
- `aarch64` 使用软链接机制，实际库目录由 `deploy.toml` 中 `lib_dir` 指定
- `run.sh` 通过 `ARCH=$(uname -m)` 自动查找对应库目录

## 9. systemd 服务 (可选)

如果 `deploy/lingbase.service` 存在，部署时会自动安装：

```ini
[Unit]
Description=Lingbase LLM Inference Service
After=network.target

[Service]
Type=simple
User=etsme
WorkingDirectory=/home/etsme/lingbase
ExecStart=/home/etsme/lingbase/run.sh
Restart=always

[Install]
WantedBy=multi-user.target
```

**部署行为:**
- 复制 `lingbase.service` 到 `/etc/systemd/system/`
- 执行 `systemctl daemon-reload`
- 执行 `systemctl enable lingbase`

## 11. 常见问题

### Q1: 远端服务器找不到 libllama.so

检查 `run.sh` 的 `LD_LIBRARY_PATH` 是否正确设置，确认 `lib/` 目录下存在对应的软链接或库文件。

### Q2: 交叉编译失败

确认已安装 `gcc-aarch64-linux-gnu` 并添加了 Rust 目标: `rustup target add aarch64-unknown-linux-gnu`。

### Q3: 部署后服务无法启动

检查 `config/environment.toml` 配置文件是否正确，以及模型文件路径是否有效。
