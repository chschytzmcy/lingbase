## 1. 项目概述

### 1.1 项目定位

Lingbase 是一个边缘 LLM 模型推理服务，核心特性：

- **轻量高效**：基于 llama.cpp，无重型依赖
- **OpenAI 兼容**：支持 `/v1/chat/completions` API
- **流式输出**：支持 SSE 实时 token 流式返回
- **异构硬件**：支持 CPU/CUDA/RKNN3 后端抽象
- **性能指标**：TTFT、ITL、Throughput、E2E、P90/P99

**关键说明**：
- `llama.cpp`、`CUDA`、`RKNN3` 是并列的硬件后端选项，各自独立实现 `LLMBackend` trait
- `llama.cpp` 后端（包括 CPU/GPU）通过 `LlamaHardware` trait 复用公共推理逻辑
- `RKNN3` 后端直接实现 `LLMBackend`，不依赖 llama.cpp

### 2.2 核心 Trait 设计

#### BaseBackend — 基础后端接口

所有后端的公共接口，与硬件解耦：

```rust
#[async_trait]
pub trait BaseBackend: Send + Sync {
    async fn init(model_config: &ModelConfig) -> Result<Self, BackendError>
    where Self: Sized;

    fn name(&self) -> &'static str;
    fn health_check(&self) -> bool;
    fn resource_usage(&self) -> Vec<ResourceType>;
}
```

#### LLMBackend — LLM 推理接口

继承 BaseBackend，提供 LLM 推理能力：

```rust
#[async_trait]
pub trait LLMBackend: BaseBackend {
    async fn inference_stream(
        &self,
        input: LLMInferenceInput,
        options: LLMInferenceOptions,
    ) -> Result<DroppableReceiver<StreamChunk>, BackendError>;

    async fn context_size(&self) -> Result<usize, BackendError>;
}
```

#### LlamaHardware — llama.cpp 硬件抽象

抽象 CPU/CUDA 等不同硬件的模型加载方式：

```rust
pub trait LlamaHardware: Send + Sync {
    fn name(&self) -> &'static str;
    fn load_model(&self, model_path: &Path, n_ctx: i32) -> InferenceResult<LlamaModelHandle>;
    fn load_tokenizer(&self, model: &LlamaModel) -> InferenceResult<Tokenizer>;
    fn vocab_size(&self, model: &LlamaModel) -> usize;
    fn load_context(&self, model_ptr: Option<ModelPtr>, n_ctx: u32, n_threads: i32) -> InferenceResult<LlamaContext>;
}
```

### 2.3 关键类型

```rust
pub enum LLMInferenceInput {
    Tokens(Vec<u32>),
    Messages(Vec<Message>),
}

pub struct LLMInferenceOptions {
    pub id: String,
    pub max_tokens: usize,
    pub temperature: f32,
    pub top_p: f32,
    pub top_k: usize,
    pub repeat_penalty: f32,
    pub stream: bool,
    pub stop: Vec<String>,
    pub enable_thinking: Option<bool>,
}

pub enum StreamChunk {
    Normal(LLMSteamChunkType),
    Finish(FinishReason),
    Error(String),
}

pub struct DroppableReceiver<T: Send> {
    receiver: Option<tokio::sync::mpsc::UnboundedReceiver<T>>,
    abort_fn: Option<Box<dyn Fn() -> Result<(), BackendError> + Send + Sync>>,
}
// Drop 时自动调用 abort_fn 终止推理
```

## 3. 后端实现

Lingbase 支持多种硬件后端，它们统一实现 `LLMBackend` trait：

| 后端 | 实现 | 说明 |
|------|------|------|
| llama.cpp (CPU/GPU) | `cpu.rs` + `runner/llama_hardware.rs` | 通过 `LlamaHardware` trait 复用公共逻辑 |
| RKNN3 | `runner_rknn3.rs` | 直接实现 `LLMBackend`，条件编译 |
| CUDA | `cuda.rs` + `runner/llama_hardware.rs` | 通过 `LlamaHardware` trait 复用公共逻辑 |

### 3.1 LlamaHardware + LlamaRunner

`llama.cpp` 后端通过 `LlamaHardware` trait 抽象硬件差异，`LlamaRunner<H>` 提供公共推理逻辑：

```rust
// 硬件抽象 - 不同硬件实现此 trait
pub trait LlamaHardware: Send + Sync {
    fn name(&self) -> &'static str;
    fn load_model(&self, model_path: &Path, n_ctx: i32) -> InferenceResult<LlamaModelHandle>;
    fn load_tokenizer(&self, model: &LlamaModel) -> InferenceResult<Tokenizer>;
    fn vocab_size(&self, model: &LlamaModel) -> usize;
    fn load_context(&self, model_ptr: Option<ModelPtr>, n_ctx: u32, n_threads: i32) -> InferenceResult<LlamaContext>;
}

// CpuHardware 示例
pub struct CpuHardware;

impl LlamaHardware for CpuHardware {
    fn name(&self) -> &'static str { "llama-cpu" }
    fn load_model(&self, model_path: &Path, _n_ctx: i32) -> InferenceResult<LlamaModelHandle> {
        let model = LlamaModel::from_file(model_path, 0)?;
        let vocab_ptr = model.vocab_ptr().ok_or(InferenceError::BackendNotInitialized)?;
        Ok(LlamaModelHandle { model, vocab_ptr: Some(vocab_ptr) })
    }
    // ...
}

// 通用 Runner，通过泛型支持不同硬件
pub struct LlamaRunner<H: LlamaHardware> {
    hardware: H,
    model_handle: Option<LlamaModelHandle>,
    context: Arc<tokio::sync::Mutex<Option<LlamaContext>>>,
    tokenizer: Arc<tokio::sync::Mutex<Option<Tokenizer>>>,
    n_ctx: u32,
    n_vocab: usize,
}

impl<H: LlamaHardware + 'static> LLMBackend for LlamaRunner<H> {
    async fn inference_stream(&self, input, options) -> Result<DroppableReceiver<StreamChunk>, BackendError> {
        // 统一的推理逻辑，hardware 提供具体实现
    }
}
```

### 3.2 CpuBackend

`src/backend/cpu.rs` - CPU 后端实现，同时实现三层 trait：

```rust
pub struct CpuBackend {
    runner: LlamaRunner<CpuHardware>,
}

// 保留向后兼容的 InferenceBackend 实现
impl InferenceBackend for CpuBackend { ... }

// 实现 BaseBackend + LLMBackend 以便在 BackendManager 中使用
#[async_trait]
impl BaseBackend for CpuBackend { ... }

#[async_trait]
impl LLMBackend for CpuBackend {
    async fn inference_stream(...) -> Result<DroppableReceiver<StreamChunk>, BackendError> {
        self.runner.inference_stream(input, options).await
    }
    async fn context_size(&self) -> Result<usize, BackendError> {
        self.runner.context_size().await
    }
}
```

### 3.3 Rknn3Backend

`src/backend/runner_rknn3.rs` - RKNN3 后端（条件编译）：

### 3.4 BackendManager

`src/backend/manager.rs` - 管理多个后端的加载、卸载、健康检查：

```rust
pub struct BackendManager {
    backends: RwLock<HashMap<String, Arc<dyn LLMBackend>>>,
    resource_capacity: Vec<ResourceType>,
}

impl BackendManager {
    pub async fn load(&self, config: &ModelConfig) -> Result<(), BackendError> {
        match config.backend_type().unwrap() {
            "llama-cpu" | "cpu" => {
                let backend = CpuBackend::new(&config.model_path, config.context_size as i32)?;
                let mut backends = self.backends.write().await;
                backends.insert(config.name.clone(), Arc::new(backend));
            }
            "rknn3" => { ... }
            _ => return Err(BackendError::UnknownBackend(...)),
        }
        Ok(())
    }

    pub async fn get(&self, name: &str) -> Option<Arc<dyn LLMBackend>> { ... }
    pub async fn unload(&self, name: &str) -> Result<(), BackendError> { ... }
    pub async fn list(&self) -> Vec<String> { ... }
    pub async fn health_check(&self, name: &str) -> bool { ... }
}
```

## 10. 扩展新硬件

新增硬件后端（如 RKNN2、NPU 等）：

1. **复制 rknn3-sys 为模板**：
   ```bash
   cp -r lib/rknn3-sys lib/xxx-sys
   ```

2. **修改 build.rs 链接新库**：
   ```rust
   PrebuiltLibBuilder::new("xxx")
       .with_lib("libxxx_api")
       .build();
   ```

3. **实现 Backend**，实现 `BaseBackend` + `LLMBackend` trait

4. **注册到 BackendManager** 的 match 语句中

**核心原则**：新增硬件后端无需修改 `traits.rs` 或其他已实现的 runner，只需实现 `LLMBackend` trait 即可。
