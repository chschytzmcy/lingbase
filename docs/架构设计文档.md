## 1. 项目概述

### 1.1 项目定位

Lingbase 是一个部署在边缘端的 LLM 模型推理服务，核心特性：

- **轻量高效**：基于 llama.cpp，无重型依赖
- **OpenAI 兼容**：支持 `/v1/chat/completions` API
- **流式输出**：支持 SSE 实时 token 流式返回
- **跨平台**：支持 x86/ARM CPU，可扩展 CUDA/RKLLM
- **性能指标**：TTFT、ITL、Throughput、E2E、P90/P99

## 3. 核心接口设计

### 3.1 HTTP API

#### 请求格式

```
POST /v1/chat/completions
Content-Type: application/json

{
  "model": "Qwen3-4B",
  "messages": [
    {"role": "user", "content": "你好"}
  ],
  "max_tokens": 256,
  "temperature": 0.7,
  "top_p": 0.9,
  "stream": false    // true 启用流式输出
}
```

#### 响应格式 (非流式)

```json
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "created": 1234567890,
  "model": "Qwen3-4B",
  "choices": [{
    "index": 0,
    "message": {"role": "assistant", "content": "..."},
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 10,
    "completion_tokens": 50,
    "total_tokens": 60
  }
}
```

**响应头** (`x-metrics`)：
```json
{
  "throughput_tokens_per_sec": 2.06,
  "time_to_first_token_ms": 2991,
  "end_to_end_latency_ms": 9242,
  "completion_tokens": 19,
  "inter_token_latency_ms": 347.3,
  "p90_latency_ms": 407,
  "p99_latency_ms": 411
}
```

#### 响应格式 (流式)

```
data: {"id":"chatcmpl-xxx","object":"chat.completion.chunk",...}
data: {"id":"chatcmpl-xxx","object":"chat.completion.chunk","choices":[{"delta":{"content":"你"}}]}
data: {"id":"chatcmpl-xxx","object":"chat.completion.chunk","choices":[{"delta":{"content":"好"}}]}
...
data: {"id":"chatcmpl-xxx","object":"chat.completion.chunk","choices":[{"finish_reason":"stop"}]}
: [METRICS] {"throughput_tokens_per_sec":6.99,"time_to_first_token_ms":514,...}
```

### 3.2 Backend Trait

```rust
pub trait InferenceBackend: Send + Sync {
    fn name(&self) -> &str;
    fn health_check(&self) -> bool;
    fn memory_stats(&self) -> MemoryStats;
    fn max_context_size(&self) -> usize;

    // 同步推理
    fn forward(&self, tokens: &[i32], config: &InferenceConfig)
        -> InferenceResult<ForwardResult>;

    // 流式推理
    fn forward_stream(&self, tokens: &[i32], config: &InferenceConfig)
        -> Pin<Box<dyn Stream<Item = InferenceResult<StreamToken>> + Send>>;

    // 文本处理
    fn tokenize(&self, text: &str) -> InferenceResult<Vec<i32>>;
    fn detokenize(&self, tokens: &[i32]) -> InferenceResult<String>;

    // Sampling
    fn sample_token(&self, logits: &[f32], config: &InferenceConfig) -> i32;
}
```

### 3.3 流式推理实现

流式推理使用 `tokio::sync::mpsc` channel 实现：

```rust
fn forward_stream(&self, tokens: &[i32], config: &InferenceConfig)
    -> Pin<Box<dyn Stream<Item = InferenceResult<StreamToken>> + Send>>
{
    let (tx, rx) = tokio::sync::mpsc::channel(16);

    tokio::task::spawn_blocking(move || {
        // Prefill
        ctx.decode(batch)?;

        // Generation loop - 每个 token 立即发送
        for each_token {
            let text = tokenizer.decode(&[token]);
            tx.blocking_send(Ok(StreamToken {
                token, text, is_first, is_done
            }));
        }
    });

    Box::pin(ReceiverStream::new(rx))
}
```

关键点：
- 使用 `spawn_blocking` 避免阻塞 Tokio runtime
- Channel buffer 16 提供 backpressure
- 每个 token 通过 `blocking_send` 立即推送

## 4. 数据结构

### 4.1 推理配置

```rust
pub struct InferenceConfig {
    pub max_tokens: usize,      // 最大生成 tokens
    pub temperature: f32,       // 温度参数
    pub top_p: f32,            // Top-p sampling
    pub top_k: i32,            // Top-k sampling
    pub repeat_penalty: f32,   // 重复惩罚
    pub timeout_ms: Option<u64>, // 超时时间
}
```

### 4.2 推理结果

```rust
pub struct ForwardResult {
    pub tokens: Vec<i32>,           // 生成的 token IDs
    pub first_token_ms: Option<u64>, // 首 token 延迟
    pub total_ms: u64,              // 总耗时
}

pub struct StreamToken {
    pub token: i32,       // Token ID
    pub text: String,     // 解码文本
    pub is_first: bool,   // 是否首 token
    pub is_done: bool,    // 是否完成
}
```

### 4.3 性能指标

```rust
pub struct StreamMetrics {
    pub throughput_tokens_per_sec: f64,  // Token 吞吐率
    pub time_to_first_token_ms: u64,      // 首 Token 时延
    pub end_to_end_latency_ms: u64,      // 端到端时延
    pub completion_tokens: usize,         // 生成 token 数
    pub inter_token_latency_ms: f64,     // Token 间延迟
    pub p90_latency_ms: u64,             // P90 分位数
    pub p99_latency_ms: u64,             // P99 分位数
}
```

    .map(|m| format!("<|im_start|>{}\n{}<|im_end|>", m.role, m.content))

prompt.push_str("<|im_start|>assistant\n/no_think\n");
